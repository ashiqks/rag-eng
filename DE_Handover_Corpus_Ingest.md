## 0. Status of what's in GCS today (read first)

The HTML files and `metadata.jsonl` currently in `gs://gap-genai-discovery-corpus-html/` (and in [`synthetic_corpus/`](synthetic_corpus/) in this repo) are **mock data** generated by [`synthetic_corpus/generate.py`](synthetic_corpus/generate.py). **DE is taking over the entire ingest** — both the HTML and the sidecar:

1. **Extract** real Test & Learn COE pages from Confluence (space `TLCOE`, plus any future spaces agreed in §8).
2. **Render** each page to HTML as described in §3.
3. **Derive** every `structData` row in `metadata.jsonl` **from the real Confluence page properties** (page-properties macro, labels, body, version metadata) — *do not* port any value from the synthetic file. Only the **shape** of the synthetic file is canonical; the **values** are throwaway.
4. **Replace** the mock contents in GCS via the cutover in §10.

Once the first real run lands and passes the §7 gates, the GenAI app team will delete the mock objects and the `synthetic_corpus/` reference data will be retained in-repo as schema documentation only.

## 2. `metadata.jsonl` contract (canonical schema)

One JSON object per line, UTF-8 **no BOM**, **LF** line endings, **no trailing newline on the last line is OK** but a trailing newline is also acceptable. Top-level shape is fixed by Discovery Engine:

```json
{
  "id": "<confluence_page_id>",
  "structData": { ... see §2.1 ... },
  "content": {
    "mimeType": "text/html",
    "uri": "gs://gap-genai-discovery-corpus-html/pages/<space>/<page_id>.html"
  }
}
```

Rules:

- **`id`** = the Confluence page ID (string). Must be globally unique within the data store; a stable `id` lets VAIS upsert rather than duplicate on re-import.
- **`content.uri`** must point to an object that exists in GCS at the moment `documents.import` runs. Upload HTML first, then write the sidecar (see §4 ordering).
- **`structData`** is a flat JSON object; nested objects allowed but keep ≤ 1 level of nesting for filterability. All values must be JSON-serialisable scalars or arrays of scalars. **No nulls** — omit the key or use `""` / `[]` instead.
- One row per HTML object. The sidecar is **replaced atomically** each run; do not append.

## 3. Per-page HTML rendering (the other half DE owns)

### 3.1 File path
`gs://gap-genai-discovery-corpus-html/pages/<space>/<page_id>.html`

### 3.2 Required `<meta>` tags inside `<head>`
Inject **one per field** (lowercase `name`, value as string). At minimum: every "required" structData field above, plus `last_updated` (Confluence `version.when`, ISO-8601 UTC) and `confluence_labels` (comma-separated; treat as low-confidence per Meeting 4 §8 — do not filter on it).

Also include a single `<script type="application/ld+json">` block whose contents are the same `structData` object as a JSON-LD payload. This is what the Exporter's HTML→JSON post-processor reads if VAIS chunking ever loses a meta tag.

### 3.3 `sha256_source`
Compute over the **canonicalised** HTML body (strip leading/trailing whitespace, normalise CRLF→LF, exclude the `<script type="application/ld+json">` block to avoid self-reference). Skip re-upload + skip the sidecar row update when the value matches the previous `ingest_state.sha256` for that `page_id`.

### 3.4 Soft-delete
If a Confluence page disappears between runs (404 on fetch, or absent from the space listing), upload an empty tombstone `pages/<space>/<page_id>.html` containing a single `<meta name="deleted" content="true">` tag **and omit it from `metadata.jsonl`**. VAIS will purge on next import.

## 4. Execution order each run (critical — do not reorder)

```
1. read   app_config.ingest_state  (last_sync_at, per-page sha256 map)
2. fetch  Confluence REST: pages where version.when > last_sync_at
3. for each page:
       render HTML  → compute sha256  → skip if unchanged
       upload       gs://...-corpus-html/pages/<space>/<page_id>.html
       buffer       { id, structData, content.uri }  into in-memory list
4. reconcile deletes (tombstone HTML, drop from buffer)
5. write  gs://...-corpus-html/metadata.jsonl.tmp-<run_id>
6. gcs-copy → metadata.jsonl   (atomic rename; old manifest never overwritten in-place)
7. upsert app_config.ingest_state  + insert ingest_runs row
8. publish Pub/Sub gap-genai-events  { subject: "corpus-ready",
                                       run_id, manifest_uri, page_count }
```

**Why HTML before sidecar:** Discovery Engine `documents.import` will fail the whole batch if a `content.uri` 404s. Writing the sidecar last guarantees every referenced HTML object exists.

## 6. Operational expectations

- **Cadence:** weekly delta (Cloud Scheduler cron `0 3 * * 1` PT). One-shot full reindex available via manual job execution with env `FULL_REFRESH=true`.
- **Runtime budget:** ~700 pages today, ~5k worst-case; target end-to-end < 15 min.
- **Telemetry (AR-5, AR-6):** emit OTel logs + metrics from the Job; the platform-wide log-based metrics already exist — DE only needs to log `page_count`, `pages_changed`, `pages_deleted`, `bytes_written`, `confluence_latency_ms`, `gcs_write_latency_ms`. Do **not** write request logs to BigQuery; logs go to Cloud Observability.
- **Alerting:** raise on (a) job failure, (b) >10 % drop in `page_count` vs previous run, (c) any page where `metadata_quality=low` after extraction.

## 7. Validation gates (must pass before declaring done)

1. **Schema validation:** every `metadata.jsonl` line parses as JSON, has the three top-level keys (`id`, `structData`, `content`), and `content.mimeType == "text/html"`.
2. **Referential integrity:** for every row, `gsutil stat <content.uri>` returns 0. No orphan rows; no orphan HTML objects (tombstones excepted).
3. **Required-field non-null check:** for every row, all ✅-marked fields in §2.1 are present and non-empty.
4. **Idempotency:** running the job twice with no Confluence changes results in **0** new GCS writes and an identical `metadata.jsonl` (byte-identical except for any `run_id`-derived tmp object names, which must not appear in the final manifest).
5. **End-to-end smoke:** trigger the corpus-ready Pub/Sub event manually with the new manifest; confirm the reindex job reports `docs_imported == page_count` and `docs_failed == 0` in `ingest_runs`.
6. **Snapshot diff vs synthetic:** for a sample of 5 real Confluence pages, the generated `structData` matches the field set in [`synthetic_corpus/metadata.jsonl`](synthetic_corpus/metadata.jsonl) (no missing keys, no extra top-level keys).

## 10. Cutover from mock data to real data

Today the bucket holds synthetic HTML + a synthetic `metadata.jsonl`. The first real run must replace both **atomically** so VAIS never sees a mixed corpus. Do it in this order:

1. **Prepare under a staging prefix** (no impact on prod data store):
   - `gs://gap-genai-discovery-corpus-html/_staging/<run_id>/pages/...`
   - `gs://gap-genai-discovery-corpus-html/_staging/<run_id>/metadata.jsonl`
2. **Run validation §7** against the staging objects (point a temporary script at the staging manifest; do NOT trigger the prod reindex).
3. **Snapshot the mock data for rollback** (one-time):
   ```
   gsutil -m mv gs://gap-genai-discovery-corpus-html/pages         gs://gap-genai-discovery-corpus-html/_mock_archive/pages
   gsutil    mv gs://gap-genai-discovery-corpus-html/metadata.jsonl gs://gap-genai-discovery-corpus-html/_mock_archive/metadata.jsonl
   ```
4. **Promote staging to the live prefix** (still no VAIS impact yet):
   ```
   gsutil -m mv gs://gap-genai-discovery-corpus-html/_staging/<run_id>/pages         gs://gap-genai-discovery-corpus-html/pages
   gsutil    mv gs://gap-genai-discovery-corpus-html/_staging/<run_id>/metadata.jsonl gs://gap-genai-discovery-corpus-html/metadata.jsonl
   ```
5. **Truncate `app_config.ingest_state`** (it currently holds mock `sha256` values that would cause every real page to be skipped):
   ```sql
   TRUNCATE TABLE `gap-genai-discovery.app_config.ingest_state`;
   ```
   …then run the exporter once with `FULL_REFRESH=true` so the table is repopulated from the real Confluence snapshot (this run only re-reads the manifest, it should not re-upload HTML because §4 step 3's sha256 check will short-circuit against the freshly-uploaded staging files — confirm zero GCS writes).
6. **Publish the corpus-ready Pub/Sub event** for the first time against real data. The GenAI app team's reindex job will issue a full `documents.import` (purge-and-replace mode for cutover, not incremental).
7. **Hold VAIS smoke tests** before deleting `_mock_archive/`. After 7 days of green production runs, delete the archive:
   ```
   gsutil -m rm -r gs://gap-genai-discovery-corpus-html/_mock_archive
   ```

**Rollback** (any time within the 7-day window): reverse step 4 using `_mock_archive/` as the source and re-issue a full reindex.

## 11. Field-derivation: Confluence → `structData`

Where each required field comes from in the real Confluence page. Source-of-truth Confluence template: Meeting 2 §10 / Meeting 3 §template-walkthrough. If a field cannot be found in the page, set `metadata_quality="low"` and **omit the field** (do not invent values).

| `structData` field | Confluence source | Extraction technique |
|---|---|---|
| `test_id` | "Test ID" row in the **Page Properties** macro at the top of the page | Page-properties REST endpoint, or XPath on rendered HTML `td[preceding-sibling::th[normalize-space()='Test ID']]` |
| `confluence_page_id` | REST `id` field on the page object | Direct |
| `confluence_url` | `https://gap.atlassian.net/wiki/spaces/<space>/pages/<id>` | Compose from REST `_links.webui` |
| `title` | REST `title` | Direct, trimmed |
| `brand`, `brand_code` | Title prefix `[ON …]`, `[BR …]`, etc., **and** the "Brand" row in page-properties | Prefer page-properties; fall back to title-prefix regex; validate against `BRANDS` list in `generate.py` |
| `market` | "Market" page-properties row, or Confluence label `market-us` / `market-ca` / `market-mx` / `market-global` | Label preferred (single source of truth) |
| `value_stream` | "Value Stream" row | Validate against `VALUE_STREAMS` allow-list |
| `channel`, `channel_section`, `device` | "Channel", "Channel Section", "Device" rows | Validate against allow-lists |
| `audience` | "Audience" row | Free-text in template — bucket into `AUDIENCES` allow-list; otherwise drop |
| `tactic` | "Tactic" row | Validate against allow-list |
| `test_type` | "Test Type" row | `Client-Side` / `Server-Side` only |
| `vendor` | "Vendor" row | Validate against `VENDORS` allow-list |
| `primary_kpi` | "Primary KPI" row | Must match `PRIMARY_KPIS` exactly — fail the row if not |
| `secondary_kpis` | "Secondary KPIs" row (comma- or newline-separated) | Split, trim, filter to `SECONDARY_KPIS` allow-list |
| `start_date`, `end_date` | "Start Date" / "End Date" rows | Parse to ISO `YYYY-MM-DD`; tolerate `MM/DD/YYYY` and `DD-MMM-YYYY` |
| `duration_days` | computed | `(end_date - start_date).days` — fail if negative |
| `sample_size_per_arm`, `mde_pct`, `confidence_pct`, `power_pct`, `exposure_pct`, `split` | "Stats" section of the template (page-properties or a child table) | Numeric parse; reject non-numeric |
| `outcome` | "Outcome" row | `Win` / `Loss` / `Flat` only (case-normalise) |
| `primary_kpi_delta_pct`, `net_rpv_delta_pct`, `aos_delta_pct` | "Results" section | Numeric; preserve sign |
| `visits_total`, `incremental_revenue_usd`, `averted_loss_usd` | "Results" section | Strip `$` and `,`; int-cast |
| `recommendation_adopted` | "Recommendation" section, "Adopted?" row | `Yes` / `No (averted loss)` / `N/A` / `Pending` |
| `related_test_ids` | "Related tests" macro or inline links to other `T-…` pages | Resolve link → `test_id`; dedupe |
| `learning_snippet` | "Learning / Recommendation" callout block (1–2 sentences) — Meeting 4 §7 | Lift verbatim; truncate at 240 chars; if absent → see open question §8.3 |
| `sha256_source` | computed | SHA-256 of canonicalised HTML body per §3.3 |
| `metadata_quality` | computed | `high` if all ✅-fields present, `medium` if ≥1 optional missing, `low` if any ✅-field missing |
| `template_version` | Confluence label `template-2017-stable` (or similar) | Label scan; default to `unknown` |
| `acl_principals` | Confluence page restrictions (read group), mapped via the table provided by PSEC | REST `/restriction/byOperation/read`; map group name → Workspace group email |

Reject (do not write to the manifest) any page that fails the required-field check; emit a row to a `ingest_rejects` BQ table with `page_id`, `missing_fields[]`, `run_id` so the COE team can fix the template at source.
