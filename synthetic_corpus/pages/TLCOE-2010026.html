1. Test Plan

1.1 Problem Statement

On Gap Factory Checkout, the current control experience does not sufficiently support checkout start rate for the all customers segment. Based on prior time savings-tactic learnings across the discount tier, we want to evaluate whether the proposed change moves the needle within the 90-day window.

1.2 Hypothesis

If we apply 'Wallet pay promotion' on Gap Factory Checkout (iOS Native) to all customers, then we expect a lift of at least 1.04% on Checkout Start Rate because promoting wallet pay above the billing form shortens checkout for mobile users.

1.3 Experimental Changes

  • Control: existing Checkout experience on Gap Factory iOS Native.
  • Challenger: Wallet pay promotion variant — promoting wallet pay above the billing form shortens checkout for mobile users.

2. Test Results

2.1 Findings (TL;DR)

Across the 90-day test window, the challenger underperformed the control on the primary KPI (Checkout Start Rate) by 1.29% (statistically significant in the wrong direction). Net RPV moved by -0.71% and AOS by -0.25%.
