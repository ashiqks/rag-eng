1. Test Plan

1.1 Problem Statement

On Gap Factory PDP, the current control experience does not sufficiently support aur for the loyalty members segment. Based on prior time savings-tactic learnings across the discount tier, we want to evaluate whether the proposed change moves the needle within the 28-day window.

1.2 Hypothesis

If we apply 'PDP default size' on Gap Factory PDP (Desktop) to loyalty members, then we expect a lift of at least 0.77% on AUR because defaulting to the most-purchased size for the current sku saves a tap.

1.3 Experimental Changes

  • Control: existing PDP experience on Gap Factory Desktop.
  • Challenger: PDP default size variant — defaulting to the most-purchased size for the current sku saves a tap.

2. Test Results

2.1 Findings (TL;DR)

Across the 28-day test window, no statistically significant difference was observed on the primary KPI (AUR (Average Unit Retail)); observed delta -0.19% sits within MDE (0.77%). Net RPV -0.28%, AOS -0.18% — also within noise.
