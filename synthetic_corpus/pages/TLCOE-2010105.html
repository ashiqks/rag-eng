1. Test Plan

1.1 Problem Statement

On Gap Account, the current control experience does not sufficiently support checkout start rate for the high-aov customers segment. Based on prior time savings-tactic learnings across the mid tier, we want to evaluate whether the proposed change moves the needle within the 90-day window.

1.2 Hypothesis

If we apply 'Auto address' on Gap Account (Tablet Web) to high-aov customers, then we expect a lift of at least 0.6% on Checkout Start Rate because auto-populating the most-used saved address speeds up checkout starts.

1.3 Experimental Changes

  • Control: existing Account experience on Gap Tablet Web.
  • Challenger: Auto address variant — auto-populating the most-used saved address speeds up checkout starts.

2. Test Results

2.1 Findings (TL;DR)

Across the 90-day test window, the challenger outperformed the control on the primary KPI (Checkout Start Rate) by 2.99% (statistically significant at the configured threshold). Net RPV moved by 2.22% and AOS by 2.26%.
