1. Test Plan

1.1 Problem Statement

On Gap PLP, the current control experience does not sufficiently support checkout start rate for the first-time buyers segment. Based on prior time savings-tactic learnings across the mid tier, we want to evaluate whether the proposed change moves the needle within the 42-day window.

1.2 Hypothesis

If we apply 'Sticky PLP filters' on Gap PLP (Mobile Web) to first-time buyers, then we expect a lift of at least 1.02% on Checkout Start Rate because a persistent filter rail removes the need to scroll back to the top, accelerating refinement and conversion.

1.3 Experimental Changes

  • Control: existing PLP experience on Gap Mobile Web.
  • Challenger: Sticky PLP filters variant — a persistent filter rail removes the need to scroll back to the top, accelerating refinement and conversion.
