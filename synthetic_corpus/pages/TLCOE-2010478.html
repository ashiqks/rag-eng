1. Test Plan

1.1 Problem Statement

On Banana Republic PDP, the current control experience does not sufficiently support checkout start rate for the all customers segment. Based on prior urgency-tactic learnings across the premium tier, we want to evaluate whether the proposed change moves the needle within the 60-day window.

1.2 Hypothesis

If we apply 'PDP scarcity label' on Banana Republic PDP (Tablet Web) to all customers, then we expect a lift of at least 1.07% on Checkout Start Rate because a scarcity label on pdp triggers urgency and lifts atb rate.

1.3 Experimental Changes

  • Control: existing PDP experience on Banana Republic Tablet Web.
  • Challenger: PDP scarcity label variant — a scarcity label on pdp triggers urgency and lifts atb rate.

2. Test Results

2.1 Findings (TL;DR)

Across the 60-day test window, the challenger outperformed the control on the primary KPI (Checkout Start Rate) by 0.61% (statistically significant at the configured threshold). Net RPV moved by 0.84% and AOS by 0.97%.
