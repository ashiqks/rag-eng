1. Test Plan

1.1 Problem Statement

On Banana Republic PDP, the current control experience does not sufficiently support pdp -> shopping bag conversion for the non-loyalty segment. Based on prior time savings-tactic learnings across the premium tier, we want to evaluate whether the proposed change moves the needle within the 150-day window.

1.2 Hypothesis

If we apply 'PDP default size' on Banana Republic PDP (Tablet Web) to non-loyalty, then we expect a lift of at least 0.71% on PDP -> Shopping Bag Conversion because defaulting to the most-purchased size for the current sku saves a tap.

1.3 Experimental Changes

  • Control: existing PDP experience on Banana Republic Tablet Web.
  • Challenger: PDP default size variant — defaulting to the most-purchased size for the current sku saves a tap.

2. Test Results

2.1 Findings (TL;DR)

Across the 150-day test window, the challenger outperformed the control on the primary KPI (PDP -> Shopping Bag Conversion) by 2.08% (statistically significant at the configured threshold). Net RPV moved by 2.28% and AOS by 0.11%.
