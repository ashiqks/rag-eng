1. Test Plan

1.1 Problem Statement

On Gap Shopping Bag, the current control experience does not sufficiently support checkout start rate for the loyalty members segment. Based on prior time savings-tactic learnings across the mid tier, we want to evaluate whether the proposed change moves the needle within the 42-day window.

1.2 Hypothesis

If we apply 'Bag auto promo' on Gap Shopping Bag (Desktop) to loyalty members, then we expect a lift of at least 1.36% on Checkout Start Rate because automatically applying the best eligible promo code lifts conversion in the bag step.

1.3 Experimental Changes

  • Control: existing Shopping Bag experience on Gap Desktop.
  • Challenger: Bag auto promo variant — automatically applying the best eligible promo code lifts conversion in the bag step.
