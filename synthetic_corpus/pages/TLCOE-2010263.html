1. Test Plan

1.1 Problem Statement

On Gap PLP, the current control experience does not sufficiently support checkout completion rate for the high-aov customers segment. Based on prior quality-tactic learnings across the mid tier, we want to evaluate whether the proposed change moves the needle within the 120-day window.

1.2 Hypothesis

If we apply 'PLP affinity sort' on Gap PLP (Desktop) to high-aov customers, then we expect a lift of at least 0.31% on Checkout Completion Rate because ml-ranked sort using prior browsing affinity should outperform default 'featured' sort.

1.3 Experimental Changes

  • Control: existing PLP experience on Gap Desktop.
  • Challenger: PLP affinity sort variant — ml-ranked sort using prior browsing affinity should outperform default 'featured' sort.
