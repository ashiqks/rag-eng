1. Test Plan

1.1 Problem Statement

On Banana Republic Checkout, the current control experience does not sufficiently support checkout start rate for the returning visitors segment. Based on prior quality-tactic learnings across the premium tier, we want to evaluate whether the proposed change moves the needle within the 90-day window.

1.2 Hypothesis

If we apply 'Checkout inline validation' on Banana Republic Checkout (Android Native) to returning visitors, then we expect a lift of at least 1.47% on Checkout Start Rate because validating fields on blur (vs on submit) reduces error-bounce.

1.3 Experimental Changes

  • Control: existing Checkout experience on Banana Republic Android Native.
  • Challenger: Checkout inline validation variant — validating fields on blur (vs on submit) reduces error-bounce.

2. Test Results

2.1 Findings (TL;DR)

Across the 90-day test window, the challenger underperformed the control on the primary KPI (Checkout Start Rate) by 4.45% (statistically significant in the wrong direction). Net RPV moved by -3.77% and AOS by 0.08%.

2.2 Variation Description

  • Control: as-is Checkout on Banana Republic Android Native.
  • Challenger: applies the Checkout inline validation treatment described in §1.3.
