1. Test Plan

1.1 Problem Statement

On Old Navy PDP, the current control experience does not sufficiently support email capture rate for the loyalty members segment. Based on prior time savings-tactic learnings across the discount tier, we want to evaluate whether the proposed change moves the needle within the 90-day window.

1.2 Hypothesis

If we apply 'PDP default size' on Old Navy PDP (iOS Native) to loyalty members, then we expect a lift of at least 1.28% on Email Capture Rate because defaulting to the most-purchased size for the current sku saves a tap.

1.3 Experimental Changes

  • Control: existing PDP experience on Old Navy iOS Native.
  • Challenger: PDP default size variant — defaulting to the most-purchased size for the current sku saves a tap.

2. Test Results

2.1 Findings (TL;DR)

Across the 90-day test window, the challenger outperformed the control on the primary KPI (Email Capture Rate) by 1.54% (statistically significant at the configured threshold). Net RPV moved by 1.98% and AOS by 1.48%.
