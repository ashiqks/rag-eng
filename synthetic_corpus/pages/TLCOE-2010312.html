1. Test Plan

1.1 Problem Statement

On Gap Shopping Bag, the current control experience does not sufficiently support add-to-bag rate for the non-loyalty segment. Based on prior time savings-tactic learnings across the mid tier, we want to evaluate whether the proposed change moves the needle within the 90-day window.

1.2 Hypothesis

If we apply 'Bag auto promo' on Gap Shopping Bag (Mobile Web) to non-loyalty, then we expect a lift of at least 1.14% on Add-to-Bag Rate because automatically applying the best eligible promo code lifts conversion in the bag step.

1.3 Experimental Changes

  • Control: existing Shopping Bag experience on Gap Mobile Web.
  • Challenger: Bag auto promo variant — automatically applying the best eligible promo code lifts conversion in the bag step.
