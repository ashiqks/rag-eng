1. Test Plan

1.1 Problem Statement

On Gap PDP, the current control experience does not sufficiently support email capture rate for the first-time buyers segment. Based on prior time savings-tactic learnings across the mid tier, we want to evaluate whether the proposed change moves the needle within the 42-day window.

1.2 Hypothesis

If we apply 'PDP default size' on Gap PDP (Mobile Web) to first-time buyers, then we expect a lift of at least 0.22% on Email Capture Rate because defaulting to the most-purchased size for the current sku saves a tap.

1.3 Experimental Changes

  • Control: existing PDP experience on Gap Mobile Web.
  • Challenger: PDP default size variant — defaulting to the most-purchased size for the current sku saves a tap.

2. Test Results

2.1 Findings (TL;DR)

Across the 42-day test window, no statistically significant difference was observed on the primary KPI (Email Capture Rate); observed delta -0.12% sits within MDE (0.22%). Net RPV 0.21%, AOS -0.12% — also within noise.
