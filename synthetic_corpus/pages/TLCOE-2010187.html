1. Test Plan

1.1 Problem Statement

On Old Navy Checkout, the current control experience does not sufficiently support pdp -> shopping bag conversion for the high-aov customers segment. Based on prior quality-tactic learnings across the discount tier, we want to evaluate whether the proposed change moves the needle within the 120-day window.

1.2 Hypothesis

If we apply 'Checkout inline validation' on Old Navy Checkout (Mobile Web) to high-aov customers, then we expect a lift of at least 0.41% on PDP -> Shopping Bag Conversion because validating fields on blur (vs on submit) reduces error-bounce.

1.3 Experimental Changes

  • Control: existing Checkout experience on Old Navy Mobile Web.
  • Challenger: Checkout inline validation variant — validating fields on blur (vs on submit) reduces error-bounce.
