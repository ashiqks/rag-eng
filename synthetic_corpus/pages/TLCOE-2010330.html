1. Test Plan

1.1 Problem Statement

On Gap Account, the current control experience does not sufficiently support add-to-bag rate for the cart abandoners segment. Based on prior time savings-tactic learnings across the mid tier, we want to evaluate whether the proposed change moves the needle within the 150-day window.

1.2 Hypothesis

If we apply 'Auto address' on Gap Account (Android Native) to cart abandoners, then we expect a lift of at least 1.34% on Add-to-Bag Rate because auto-populating the most-used saved address speeds up checkout starts.

1.3 Experimental Changes

  • Control: existing Account experience on Gap Android Native.
  • Challenger: Auto address variant — auto-populating the most-used saved address speeds up checkout starts.
