1. Test Plan

1.1 Problem Statement

On Gap Account, the current control experience does not sufficiently support opv conversion rate for the non-loyalty segment. Based on prior time savings-tactic learnings across the mid tier, we want to evaluate whether the proposed change moves the needle within the 120-day window.

1.2 Hypothesis

If we apply 'Auto address' on Gap Account (Mobile Web) to non-loyalty, then we expect a lift of at least 0.7% on OPV Conversion Rate because auto-populating the most-used saved address speeds up checkout starts.

1.3 Experimental Changes

  • Control: existing Account experience on Gap Mobile Web.
  • Challenger: Auto address variant — auto-populating the most-used saved address speeds up checkout starts.
