1. Test Plan

1.1 Problem Statement

On Athleta Checkout, the current control experience does not sufficiently support upt for the all customers segment. Based on prior quality-tactic learnings across the premium tier, we want to evaluate whether the proposed change moves the needle within the -182-day window.

1.2 Hypothesis

If we apply 'Checkout inline validation' on Athleta Checkout (Mobile Web) to all customers, then we expect a lift of at least 0.55% on UPT because validating fields on blur (vs on submit) reduces error-bounce.

1.3 Experimental Changes

  • Control: existing Checkout experience on Athleta Mobile Web.
  • Challenger: Checkout inline validation variant — validating fields on blur (vs on submit) reduces error-bounce.
