1. Test Plan

1.1 Problem Statement

On Banana Republic PDP, the current control experience does not sufficiently support pdp -> shopping bag conversion for the high-aov customers segment. Based on prior time savings-tactic learnings across the premium tier, we want to evaluate whether the proposed change moves the needle within the 120-day window.

1.2 Hypothesis

If we apply 'PDP default size' on Banana Republic PDP (Mobile Web) to high-aov customers, then we expect a lift of at least 1.12% on PDP -> Shopping Bag Conversion because defaulting to the most-purchased size for the current sku saves a tap.

1.3 Experimental Changes

  • Control: existing PDP experience on Banana Republic Mobile Web.
  • Challenger: PDP default size variant — defaulting to the most-purchased size for the current sku saves a tap.

2.1 Findings (TL;DR)

Across the 120-day test window, no statistically significant difference was observed on the primary KPI (PDP -> Shopping Bag Conversion); observed delta -0.04% sits within MDE (1.12%). Net RPV 0.08%, AOS -0.07% — also within noise.
